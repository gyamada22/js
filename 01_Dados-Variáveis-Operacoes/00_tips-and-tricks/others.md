Como computadores realmente funcionam
Não precisa virar engenheiro de hardware, mas entender binário, como a CPU executa instruções e o que é um processo vs thread muda como você lê qualquer código. O curso "Nand to Tetris" é gratuito e te constrói um computador do zero em abstração — é um dos projetos educacionais mais bem feitos que existem.

Algoritmos e estruturas de dados fora do LeetCode
O LeetCode te ensina a resolver, mas não te ensina a pensar. Lê sobre Big O notation de verdade — não para decorar, mas para desenvolver intuição sobre por que um loop dentro de outro é perigoso em escala. O livro "Grokking Algorithms" é visual, curto e direto.

Como bancos de dados funcionam por dentro
Você já sabe SQL, mas provavelmente nunca pensou em como um banco decide executar uma query. Pesquisa query planner e índices B-tree. Entender isso te faz escrever queries melhores e te diferencia muito em entrevistas de backend.

Sistemas distribuídos de forma acessível
Parece avançado mas os conceitos básicos são simples e aparecem em toda entrevista sênior. Pesquisa CAP theorem — a ideia de que um sistema distribuído não pode ter consistência, disponibilidade e tolerância a partição ao mesmo tempo. O blog da Cloudflare tem artigos excelentes escritos para desenvolvedores.

O que acontece quando seu código vai para produção
CI/CD, Docker, o que é um container, o que é um servidor, diferença entre IaaS e PaaS. Não precisa configurar nada agora, mas entender o ciclo — código no GitHub, pipeline roda testes, deploy automático — te faz escrever código pensando em como ele vai viver fora da sua máquina.

Leitura de papers e RFCs
RFC é o documento que define como protocolos funcionam. A RFC do HTTP, a do WebSocket, a do JWT — são os documentos originais que definem o que todo mundo implementa. Lê a introdução de uma RFC só para ver como especificações técnicas são escritas. É uma habilidade rara em devs junior.

Como grandes empresas arquitetam sistemas
Os engenharia blogs da Airbnb, Netflix, Uber e Shopify são públicos e gratuitos. Eles descrevem problemas reais que você vai reconhecer — "nossa API estava lenta, fizemos isso" — escritos por pessoas que constroem o que você usa todo dia. É motivador e educativo ao mesmo tempo.

Teoria de compiladores no nível certo
Não o curso inteiro, só o suficiente para entender o que um compilador faz. Isso explica TypeScript, Babel, ESLint e qualquer ferramenta que transforma código em código. O projeto "make a lisp" no GitHub te guia para construir um interpretador minúsculo — é um dos exercícios mais reveladores que existem para um desenvolvedor.

UX e design básico para devs
Não para virar designer, mas para parar de construir interfaces que funcionam mas são ruins de usar. Lê "The Design of Everyday Things" do Don Norman — é sobre produto em geral, não sobre software, mas vai mudar como você pensa em interfaces. "Refactoring UI" dos criadores do Tailwind é mais direto e focado em dev.

Como o mercado de tecnologia funciona
Lê sobre o que diferencia uma startup de uma big tech em termos de stack e cultura de engenharia. Pesquisa o que é technical debt, como empresas acumulam e pagam. Entender o contexto de negócio por trás das decisões técnicas é o que separa quem executa de quem propõe soluções.

Filosofia de software
"The Pragmatic Programmer" é um livro de 1999 que ainda é citado toda semana. Não é sobre linguagem nenhuma — é sobre como pensar como um profissional de software. Lê um capítulo por semana enquanto faz o curso. O "Manifesto Ágil" original tem 4 valores e 12 princípios, leva 10 minutos para ler e está por trás de como quase todo time de tech trabalha hoje.

Segurança além do básico
O projeto OWASP Top 10 lista os dez ataques mais comuns em aplicações web. É um documento curto, direto e atualizado. Depois disso pesquisa penetration testing conceitual — não para atacar nada, mas entender como um atacante pensa te faz escrever código defensivo naturalmente.

O que é observabilidade
Logs, métricas e traces — a trindade de quem mantém software em produção. Pesquisa o que é OpenTelemetry e por que empresas investem tanto em observabilidade. Quando você construir sua primeira API real vai querer saber o que está acontecendo dentro dela.
